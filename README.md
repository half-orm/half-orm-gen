# half-orm-litestar

A [halfORM](https://github.com/collorg/halfORM) extension that generates a
[Litestar](https://litestar.dev) REST API from your halfORM project.

Two complementary modes:

- **`@api_*` decorators** — hand-craft any route directly on a relation method.
- **Auto-CRUD** — declare a `CRUD_ACCESS` dict in a relation module and get
  full CRUD routes generated automatically, with per-role field access control.

## Installation

```bash
pip install half-orm-litestar
```

Requires `half-orm-dev`:

```bash
pip install half-orm-dev
```

---

## Quick start — `@api_*` decorators

Decorate methods on halfORM relation classes with the HTTP verb you want to
expose:

```python
# myproject/actor/user.py
from half_orm_litestar import tools

class User(MODEL.get_relation_class('actor.user')):

    @tools.api_get('/user/{id: uuid}', guards=['public'])
    async def get_user(self, request: "Request"):
        return self.load_user(request.path_params['id'])

    @tools.api_post('/users', guards=['connected'])
    async def create_user(self, request: "Request"):
        data = await request.json()
        ...
```

Available decorators: `api_get`, `api_post`, `api_put`, `api_patch`, `api_delete`.
All accept the same arguments as the corresponding Litestar decorators.

## Quick start — auto-CRUD

In the developer space of a half-orm-dev generated module (between `#>>>` and
`#<<<`), declare a `CRUD_ACCESS` dict at module level:

```python
# myproject/blog/post.py  (generated by half-orm-dev)

#>>> PLACE YOUR CODE BELOW THIS LINE. DO NOT REMOVE THIS LINE!

CRUD_ACCESS = {
    "GET":    {"public": ["id", "title", "published"], "connected": None},
    "POST":   {"connected": {"in": ["title", "content", "author_id"]}},
    "PUT":    {"connected": {"in": ["title", "content", "published"]}},
    "DELETE": {"admin": None},
}

#<<< PLACE YOUR CODE ABOVE THIS LINE. DO NOT REMOVE THIS LINE!
```

`half_orm litestar generate` will then produce full CRUD routes for that
relation — `GET /v0/blog/post`, `GET /v0/blog/post/{id: uuid}`,
`POST /v0/blog/post`, etc. — with access control enforced at runtime and
documented in the OpenAPI schema.

See [docs/crud_access.md](docs/crud_access.md) for the full reference.

## Generate and run

```bash
# from the root of your half-orm-dev project
half_orm litestar generate
half_orm litestar run --reload
```

`generate` scans all relation classes, writes `api/app.py`, and scaffolds
missing files in `api/` (existing files are never overwritten).

## Project structure

```
myproject/
├── api/
│   ├── __init__.py
│   ├── app.py                           ← generated, do not edit
│   ├── guards.py                        ← authentication guards (edit this)
│   ├── roles/
│   │   └── core.py                      ← @authorize_and / @authorize_or
│   └── custom/
│       ├── __init__.py
│       ├── routes.py                    ← hand-written extra routes
│       └── middlewares/
│           ├── __init__.py              ← extra middlewares list
│           └── authorization.py        ← authentication middleware (edit this)
```

Files in `api/` (except `app.py`) are created once and never overwritten.

## Customisation

### Guards (`api/guards.py`)

Two guards are provided: `public` (allow all) and `connected` (require
authenticated user). Add project-specific guards here:

```python
async def has_user_access(
    connection: ASGIConnection, handler: BaseRouteHandler = None
) -> None:
    if not connection.user:
        raise HTTPException(status_code=401)
    if connection.user != str(connection.path_params.get('user_id')):
        raise HTTPException(status_code=403)
```

### Authorization middleware (`api/custom/middlewares/authorization.py`)

Implement `Authorization` to decode tokens and populate `connection.user`.
When the file exists it is placed first in the middleware stack automatically.

```python
class Authorization(AbstractAuthenticationMiddleware):
    async def authenticate_request(self, connection: ASGIConnection) -> AuthenticationResult:
        token = connection.headers.get("Authorization", "").removeprefix("Bearer ").strip()
        try:
            payload = jwt.decode(token, SECRET, algorithms=["HS256"])
            return AuthenticationResult(user=payload["sub"], auth=token)
        except Exception:
            return AuthenticationResult(user=None, auth=None)
```

### Extra routes and middlewares

Add hand-written Litestar handlers in `api/custom/routes.py` and extra
middlewares in `api/custom/middlewares/__init__.py`.

## Advanced: programmatic use

```python
from half_orm_litestar.generate import GenApi

GenApi(
    relation_classes=my_classes,   # iterable of (RelationClass, type) pairs
    module_name='mydb',
    base_dir='/path/to/project',
)
```
